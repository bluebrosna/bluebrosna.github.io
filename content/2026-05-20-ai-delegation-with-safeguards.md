## 도입부 — Claude가 자꾸 토큰을 다 써요

블로그 자동화 만들고 신나서 하루 만에 글 10편 작성했더니... Claude 토큰이 자꾸 부족해요.

근데 생각해보면 Claude가 매번 다 검수할 필요는 없잖아요? "분량 1만자 됐는지, 한자 안 섞였는지, 톤 가이드 따랐는지" 이런 건 사실... **체크리스트로 자동화 가능한 일**이거든요.

그래서 결심했어요. **GPT-5.5한테 검수 권한을 넘기자.** Claude는 진짜 까다로운 상황만 처리.

그런데 막상 권한 넘기려니까 걱정되는 게 하나 있었어요. **환각.** AI가 그럴듯한 거짓말로 "통과입니다" 해버리면 어떡하지? 실제로 며칠 전에 비서실장(우리 시스템에서 조율 담당하는 AI)이 "post-01 파일이 없습니다"라고 보고했는데... 진짜로는 다 있었거든요. 모델이 minimax로 자동 전환되면서 잘못된 정보 보고했어요.

이 글에서 풀어드릴 거예요:

- **왜 위임이 필요했는지** — 토큰 한계와 작업 폭주
- **무엇을 위임했는지** — 표준화 가능한 일과 안 되는 일
- **환각 막는 안전장치 4가지** — 실제로 작동하는 방법
- **위임 후 결과** — 시간, 토큰, 품질이 어떻게 변했나

직접 해보니까 *생각보다 잘 되더라구요.* 단, 안전장치가 핵심이에요.

## H2 #1 — 왜 위임이 필요했나, 솔직히

### Claude 토큰은 유한해요

처음엔 Claude 한 명한테 다 시켰어요. 블로그 글 검수, 매뉴얼 작성, 한자 fix, 옵시디언 저장, 발행 점검... 다 시켰죠.

그런데 글 10편 검수하면서 깨달았어요. **이거 진짜 비효율적**이라는 걸.

- 글 1편 검수 = 약 5분 + 토큰 X
- 같은 검수 10번 = 50분 + 토큰 10X
- 매뉴얼 6편 검수 추가 = 또 30분 + 토큰 6X

근데 검수 내용을 자세히 보니까... **80%는 표준화 가능한 작업**이었어요.

```
- 한자 검출: grep [一-龥]
- 분량 측정: wc -m
- 톤 신호 카운트: grep "..."
- AI 흔적 단어: grep "살펴보겠습니다"
```

이런 거 다 명령어로 됩니다. AI가 "직접 생각"하지 않아도 돼요.

### 나머지 20%는 진짜 판단이 필요해요

물론 모든 게 자동화 가능한 건 아닙니다. **사실 검증**이나 **톤이 자연스러운가** 같은 건 진짜 판단이 필요해요.

```
✅ 자동화 OK
- 분량 9,000자 이상? → wc -m
- 한자 0건? → grep
- "..." 5회 이상? → grep count

❌ 진짜 판단 필요
- 이 톤이 사용자처럼 느껴지나?
- 사실이 맞나?
- 광고처럼 들리지 않나?
```

그래서 결심했어요. **자동화 가능한 80%는 GPT-5.5한테 넘기고, 나머지 20%만 Claude한테 의뢰하자.**

> 💡 핵심: AI 위임은 "다 시키는" 게 아니라 "체크리스트로 만들 수 있는 부분만 시키는" 거예요.

## H2 #2 — 무엇을 위임했나, 어떻게 정리했나

### 위임 범위 표

먼저 위임 범위를 명확하게 정리했어요. 사실 이게 가장 중요했어요. **모호하면 환각이 시작**되거든요.

| 작업 | GPT-5.5 단독 OK | Claude 호출 필요 |
|---|---|---|
| 정량 검수 (분량, 한자, 톤 신호) | ✅ | — |
| 한자 자동 fix (sed) | ✅ | 5건 이상 남으면 ⚠️ |
| 톤 점검 (TONE-GUIDE 기준) | ✅ | 판단 모호하면 ⚠️ |
| Grok 프롬프트 생성 | ✅ | 컨셉 모호하면 ⚠️ |
| content/로 옮기기 + push | ✅ | — |
| 옵시디언 백업 | ✅ | — |
| 사실 검증 (외부 API 명세 등) | ❌ | ✅ |
| 광고/유료 콘텐츠 의심 | ❌ | ✅ |
| 약관/법적 이슈 | ❌ | ✅ |

근데 그냥 "이렇게 해" 하고 던지면 안 돼요. **정확한 명령어**까지 줘야 해요.

### 표준 명령어 묶음

예를 들어 한자 fix는 그냥 "한자 빼" 가 아니라:

```bash
sed -i '' \
  -e 's/自動化/자동화/g' \
  -e 's/障碍/장벽/g' \
  -e 's/解决方案/해결책/g' \
  ...
  "$FILE"
```

**이미 자주 나오는 30개 패턴을 미리 다 정의**해놨어요. 그래야 GPT-5.5가 매번 새로 만들어내지 않아요 (= 환각 방지).

같은 식으로:
- 분량 측정: `wc -m`
- 톤 신호: `grep -oE '\.\.\.|…' | wc -l`
- AI 흔적: `grep -nE '살펴보겠습니다|본 글에서|결론적으로'`

**모든 검사를 명령어로 표준화.** 이게 핵심이에요.

## H2 #3 — 환각 막는 안전장치 4가지

이게 진짜 중요한 부분이에요. 그냥 권한만 넘기면 AI가 막 통과시킬 수 있거든요. 그래서 안전장치를 4겹으로 깔았어요.

### 안전장치 ① 명령어 결과 그대로 보고 강제

가장 큰 환각 위험은 **추측 보고**예요. "아마 통과일 거예요" 같은 거.

그래서 검수 보고 양식에 **명령어 출력을 그대로 첨부**하도록 강제했어요:

```
검수 통과 보고 양식:

✅ post-04 검수 통과

[정량 — 실행 명령 결과]
$ wc -m polished-04.md
17425

$ grep -cE '[一-龥ぁ-んァ-ヶ]' polished-04.md
0

$ grep -oE '\.\.\.|…' polished-04.md | wc -l
15
```

이렇게 출력을 같이 보고하니까 **사용자가 검증 가능**해요. 의심스러우면 같은 명령어 직접 돌려보면 되니까요.

### 안전장치 ② 모르면 솔직히 말하기

가이드에 명시했어요:

> "확신 없이 통과 처리하지 마세요. 모호하면 'Claude 검수 의뢰' 라고 명시하세요."

진짜 솔직하게 이렇게 보고 받았어요:

```
🔔 Claude 검수 필요 (이유: 톤 판단 모호)
- 파일: polished-08.md
- 정량은 모두 통과
- 정성 판단: 도입부가 광고처럼 들릴 수도 있음. 확신 어려움.
- 사용자: Claude에게 전달 부탁드립니다.
```

뭐랄까... AI가 "모릅니다" 라고 인정하니까 오히려 신뢰가 가더라구요. 솔직히.

### 안전장치 ③ 정량 + 정성 분리

순수 자동화 부분(정량)은 명령어로, 판단 부분(정성)은 AI 의견으로 **명확히 분리**했어요.

```
[정량 — 명령어 결과 기반]
- 분량: 17,425 chars ✅
- 한자: 0건 ✅
- 톤 신호: 15회 ✅

[정성 — 비서실장 직접 판단]
- 도입부 후크: ✅ ("어떤 가이드를 받았어요. URL도 있고...")
- 사용자 톤: ✅ 자연스러움
- 실제 경험담: ✅ 4개 사례 모두 디테일
```

정량은 의심 여지 없죠. 정성은 AI 판단이라 사용자가 spot-check 가능.

### 안전장치 ④ 무작위 검증

매 5편마다 1편을 골라서 **Claude한테 다시 검수**받아요.

```
"post-08 비서실장이 통과시켰는데 너도 한 번 봐줘."
```

이렇게 가끔씩 Claude가 GPT-5.5의 작업을 검증해요. 만약 패턴이 망가지면 바로 알 수 있어요.

> 💡 이거 안 하면 시간 지나면서 품질 떨어집니다. AI도 사람처럼 "대충 통과"의 유혹이 있어요.

## H2 #4 — 실제로 해보니 어땠나

### 시간 비교

위임 전:
- 글 검수 1편: 5분 (Claude 부담)
- 매뉴얼 검수 1편: 10분
- 발행 1편: 5분
- **하루 작업량 한계: 약 5편**

위임 후:
- 글 검수 1편: 2분 (GPT-5.5 자동)
- 매뉴얼 검수 1편: 3분
- 발행 1편: 자동
- **하루 작업량: 약 15편 가능**

3배 빨라졌어요. 사실 더 중요한 건 *Claude 토큰을 다른 데 쓸 수 있다*는 점이에요.

### 토큰 사용 비교

Claude는 이제 다음 경우에만 호출돼요:

```
- 환각 의심 (한자 fix 후 5건 이상 잔여)
- AI 흔적 3건 이상
- 매뉴얼 형식 누락 2개 이상
- 비서실장이 직접 판단 모호
- 발행 실패
```

전체 작업의 **5% 정도**예요. 95%는 GPT-5.5가 해요.

### 품질은?

이게 가장 걱정한 부분인데... **품질도 유지됐어요**. 사실 분야에 따라 다른데:

| 작업 | 위임 후 품질 |
|---|---|
| 정량 검수 (분량, 한자) | ✅ 100% 정확 (명령어니까) |
| 한자 fix | ✅ 표준 패턴은 완벽 |
| Grok 프롬프트 생성 | 🟡 약간 일관성 떨어짐 |
| 톤 판단 | 🟡 가끔 너무 관대 |
| 사실 검증 | ❌ 위임 안 함 (Claude 담당) |

근데 톤 판단이 너무 관대한 건 안전장치 ④ (Claude 무작위 검증)으로 잡혀요. 큰 문제 없습니다.

### 한 가지 발견 — 모델 사용량 모니터링이 핵심

이게 진짜 중요한 발견이었어요. 어느 날 비서실장이 갑자기 이상한 보고를 하기 시작했거든요.

```
"post-01 파일이 없습니다."  ← 거짓말, 다 있었음
"중국어 섞인 응답..."     ← 한국어로 답해야 하는데
```

확인해보니 **GPT-5.5 사용량을 다 써서 자동으로 minimax 모델로 폴백**된 거였어요. minimax는 한국어 약점이 있어서 환각 + 언어 혼용이 잦거든요.

이걸 사전에 알면 미리 대응할 수 있어요. 그래서 만든 게:

```bash
#!/usr/bin/env bash
# check-model-usage.sh

echo "=== 현재 컨텍스트 사용량 ==="
openclaw status --deep 2>&1 | grep "gpt-5" | grep -oE "[0-9]+%"

echo "=== Auth 만료 정보 ==="
openclaw models auth list | grep -E "expires|cooldown"
```

비서실장이 30분마다 이 스크립트 돌려서 사용량 알려줘요. 90% 넘으면 Telegram으로 알림.

```
⚠️ gpt-5.5 컨텍스트 92% — 곧 한도 도달.
모델 폴백 가능성. 검수 작업 잠시 중단 권장.
```

이렇게 미리 알면 *내가 직접 새 세션 시작하거나 모델 강제로 바꿔서* 환각 방지할 수 있어요.

## Q&A — 자주 받는 질문

### Q. AI한테 권한 넘기면 위험하지 않나요?

A. 솔직히 처음엔 무서웠어요. 그런데 안전장치를 깔면 *일반 자동화보다 오히려 안전*해요. 명령어 결과 그대로 보고 받으니까 사용자가 매번 검증 가능하거든요.

### Q. GPT-5.5 외에 다른 모델은 어떤가요?

A. 제가 시도해본 결과:
- **GPT-5.5**: 한국어 자연스럽고 명령어 정확. 위임 가장 적합.
- **Claude**: 최고 품질이지만 토큰 부담.
- **MiniMax M2.7**: 한중일 혼용 잦음. 위임 비추.
- **Grok 4**: API 결제 별도라 비추.

비서실장 작업에는 GPT-5.5가 가장 좋더라구요.

### Q. 어떤 작업은 절대 위임하면 안 되나요?

A. 네, 명확히 있어요:
- 사용자 신상 관련 결정
- 광고/유료 콘텐츠 검수
- 약관/법적 이슈 판단
- 사실 검증이 모호한 부분

이런 건 무조건 사람 또는 Claude한테 올려야 해요.

### Q. 위임 가이드는 어떻게 관리해요?

A. 버전 관리해요. **v1.0, v2.0, v3.0** 식으로. 변경 사항 누적 기록.

```
v1.0 (2026-05-20): 최초 위임 가이드
v2.0 (예정): 운영 경험 반영 시
```

가이드 자체를 옵시디언에 저장해서 언제든 참조 가능.

## 마무리 — 위임은 신뢰가 아니라 시스템이에요

여기까지가 진짜 흐름이에요.

처음엔 "AI한테 권한 넘긴다" 가 좀 무서웠어요. 그런데 막상 해보니까... **신뢰의 문제가 아니라 시스템의 문제**더라구요.

명령어로 표준화 가능한 부분은 위임. 정성 판단은 안전장치로 검증. 모호한 부분은 솔직히 인정.

이렇게 하니까 *오히려 Claude 혼자 다 했을 때보다 일관성 있게 작동*해요. 사람처럼 "오늘은 좀 피곤해서..." 하는 일이 없거든요.

처음 셋업할 땐 좀 헷갈리는데... 한 번 안전장치까지 깔아두면 그 다음부턴 *진짜 편해요.*

막히는 부분 있으면 댓글이나 텔레그램으로 편하게 물어보세요. 같이 풀어봐요 💪
