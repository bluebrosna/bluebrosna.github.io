## 도입부

유튜브 긴 영상의 하이라이트만 잘라내서 인스타 릴스로 올리고 싶은데, 편집 앱 열기 귀찮으시죠? **ffmpeg이면 30초 만에 됩니다.**

처음엔 저도 이걸 굳이 명령어로 해야 하나 싶었습니다. 그런데 반복 작업이 쌓이면 생각이 바뀌더라구요.

사실 이걸 알아가기까지 좀 걸렸어요. 처음엔 Premiere Pro나 키네마스터 같은 편집 앱으로 일일히 자르고 있었거든요. 10분짜리 영상을 1분 컷으로 만들려고 앱 열고 → 영상 열고 → 시간대 찾아서 → 크롭하고 → 내보내고... 이 짓을 매일 반복하니까 진짜 피곤해졌어요.

그러다 지루해서 찾은 게 ffmpeg예요. 뭐, 명령어이고 뭔가 있어 보이지만... 솔직히 처음엔 거부감 있었어요. "이게 뭐 하는 건데? 제가 이걸 왜 써요?" 하는 생각이 첫 반응이었거든요.

근데 실제로 써보니까 그게 그거더라구요. 명령어 한 줄이면 편집 앱 10분짜리 작업이 30초로 끝나니까. 사실... 그것도 한 번 해보고 나서 알게 된 거예요.

가로 영상(16:9)에 아래쪽/위쪽 빈 공간을 블러 처리하고, 중앙을 세로 비율로 크롭해서 자막까지 입히는 것 — 이 세 가지를 **ffmpeg 한 줄 명령어**로 처리하는 방법을 정리했습니다.

## 왜 ffmpeg인가? ⚡

ffmpeg를 쓰면 좋은 이유를 정리하면 이런 표가 나와요:

| 편집 앱             | 비용     | 자동화 | 처리 속도 |
| ------------------- | -------- | ------ | --------- |
| 프리미어/키네마스터 | 유료     | ❌     | 보통      |
| **ffmpeg**          | **무료** | **✅** | **빠름**  |
| 온라인 변환기       | 무료     | ❌     | 느림      |

ffmpeg는 스크립트에 넣으면 매일 같은 변환을 반복할 수 있고, macOS에 이미 설치되어 있습니다. `brew install ffmpeg`로 금방 깔려요.

그런데 사실... 이 표만 보면 ffmpeg가 항상 정답인 것처럼 보이는데, 꼭 그렇지만은 않아요. 자막을 예쁘게 넣는다거나 여러 영상 클립을 합친다거나 할 때는 편집 앱이 나을 때도 있어요.

솔직히 말하면, ffmpeg는 "단순한 변환"에는 최고지만, "복잡한 편집"에는 편집 앱이 나아요. 이 구분이 중요해요.

핵심은 **간단한 변환은 ffmpeg로, 복잡한 편집은 편집 앱으로** — 이게 현실적인 용법이죠.

### ffmpeg 설치 확인

이미 설치되어 있을 수 있어요. 터미널에서 확인해보세요:

```bash
ffmpeg -version
```

만약 "command not found"가 뜨면 설치가 필요해요:

```bash
brew install ffmpeg
```

ffmpeg가 지원하는 포맷이 엄청 많아요. mp4, mov, avi, mkv, webm... 웬만한 건 다 돼요. 심지어 이미지 시퀀스에서 영상 만들기도 되고, 영상에서 이미지 추출도 돼요.

## 핵심 1: 블러 배경 만들기

가로 영상의 빈 공간을 채우는 가장 예쁜 방법은 **좌우 또는 상하를 블러 처리**하는 것입니다.

```bash
# 상하 블러 + 세로 크롭 (16:9 → 9:16)
ffmpeg -i input.mp4 \
  -vf "crop=iw:720:0:(ih-720)/2,boxblur=15:15" \
  -c:v libx264 -preset fast -crf 23 \
  -c:a copy \
  output_blur.mp4
```

`boxblur=15:15`는 수평/수직 블러 강도입니다. 숫자가 클수록 뭉개져요.

근데 여기서 중요한 포인트... 이 명령어 그대로 복붙하면 잘 안 되는 경우가 있어요. 이유는 입력 영상의 해상도에 따라 crop 값이 달라지거든요. 그래서 계산이 필요해요.

예를 들어보자면:

- 입력: 1920×1080 (16:9)
- 세로로 만들려면 높이를 기준으로 너비를 계산: 720 × (16/9) = 1280
- 그런데 1920×1080에서 너비 1280으로 중앙 크롭하면 좌우가 너무 많이 잘려요

그래서 보통은 블러 처리하면서 크롭하는 게 나아요. 영상이 다 보존되면서 빈 공간만 채워지니까.

### 블러 배경의 원리

이게 왜 동작하는지 이해하면 더 잘 쓸 수 있어요.

ffmpeg의 비디오 필터 체인은 왼쪽에서 오른쪽으로 동작해요:

```
입력 영상 → crop 필터 → boxblur 필터 → 출력 영상
```

`crop=iw:720:0:(ih-720)/2`는:

- `iw`: 입력 영상의 너비 (전체 너비 사용)
- `720`: 출력 높이 (720픽셀)
- `0`: 잘라낼 x 오프셋
- `(ih-720)/2`: 세로 중앙 정렬을 위한 y 오프셋

그러니까 원본 영상의 세로 중앙 720픽셀 부분을 크롭하는 거예요. 1080p 영상이라면 상하에서 각각 180픽셀씩 잘라내는 거죠.

## 핵심 2: 중앙 크롭만으로 빠르게

배경은 필요 없고 **중앙 내용물만 자르고 싶다면** 이 명령어만으로 충분합니다:

```bash
# 16:9 → 9:16 중앙 크롭 (720p 기준 405x720)
ffmpeg -i input.mp4 \
  -vf "crop=405:720:(in_w-405)/2:0" \
  -c:v libx264 -preset fast -crf 23 \
  -c:a copy \
  output_cropped.mp4
```

720p 세로 기준 너비 405px을 계산한 것. 다른 해상도라면 비율만 맞추면 됩니다.

**계산 공식:**

```
세로 너비 = 세로 높이 × (9/16)
예: 1080p → 1080 × 9/16 = 607px
```

그런데 사실 이 공식만 외우고 있으면 좋겠지만... 매번 계산하기 귀찮잖아요. 그래서 보통 720p 기준으로 405×720으로 고정해서 쓰면 편해요.

근데 여기서 문제가 하나 있어요. 원본 영상의 좌우를 자르는 거라, 양쪽 중요한 내용이 잘릴 수 있어요. 예를 들어 자막이 좌우에 있으면 같이 잘려버리거든요. 그래서 **자막 위치**가 중요해요.

### 중앙 크롭의 함정

사실 중앙 크롭 방식에는 치명적인 약점이 있어요. 양쪽이 짤린다는 거예요.

유튜브 영상을 예로 들면:

- 중앙에는 주요한 내용 (사람, 텍스트)
- 양옆에는 브랜드 워터마크나 자막
- 중앙 크롭하면 중앙 중심 콘텐츠만 남음

근데 문제는... 유튜브 쇼츠나 릴스에서는 좌우 자막을 넣는 경우가 많거든요. 그렇게 되면 중앙 크롭 방식이 불리해요. 자막이 짤리니까요.

그래서 저는 이렇게 구분해요:

| 상황                | 추천 방식             |
| ------------------- | --------------------- |
| 좌우 자막 없는 영상 | 중앙 크롭 (가장 간단) |
| 좌우 자막 있는 영상 | 블러 배경 (전체 보존) |
| 배경까지 예쁘게     | 블러 배경 + 크롭 조합 |

## 핵심 3: Whisper로 자막 자동 생성 후 번인

자막이 없으면 영상만으로는 내용이 부족해 보입니다. Whisper CLI로 음성에서 자막을 자동 생성하고 ffmpeg로 번인해 보겠습니다.

### Whisper 설치

```bash
# pip로 설치
pip install whisper

# 또는 brew로 설치 (Intel Mac)
brew install whisper

# Apple Silicon Mac에서는 이렇게 (CUDA 대신 MPS)
pip install whisper
```

Whisper은 OpenAI에서 만든 음성 인식 모델이에요. 무료고, 한국어 인식도 잘 돼요.

### 자막 생성

```bash
# 1) 자막 생성 (small 모델, CPU 기준 3~5배속)
whisper input.mp4 --model small --output_format srt

# GPU가 있다면 medium 모델이 더 정확해요
whisper input.mp4 --model medium --output_format srt
```

생성된 `input.srt` 파일을 ASS 형식으로 변환하면 스타일(자막 크기, 색상, 위치)을 조절할 수 있습니다:

```bash
# 2) SRT → ASS 변환 (스타일 유지, 위치 조정 가능)
ffmpeg -i input.srt subtitle.ass
```

```bash
# 3) 자막 번인
ffmpeg -i input.mp4 -vf "ass=subtitle.ass" \
  -c:v libx264 -preset fast -crf 23 \
  -c:a copy \
  output_with_subs.mp4
```

> ⚠️ **자막 파일 경로에 한글 포함 시 인식 안 되는 경우가 있습니다.** 파일명을 영어로 지정하세요.

사실 이 부분에서 가장 많이 헤맸어요. 자막 파일 경로에 한글이 포함되어 있으면 ffmpeg가 자막을 인식 못 하더라구요. "파일 경로 한글로 되어 있는데 왜 안 되지?" 하신 분 계실 거예요. 저도 그랬거든요.

그래서 파일명을 영어로 바꿔서 실행하니까 바로 됐어요.

### Whisper 한국어 인식 품질

솔직히 말하면, Whisper의 한국어 인식 품질은 생각보다 괜찮아요. 단, 몇 가지 점이 있어요:

- **small 모델**: 속도 빠름, 한국어 정확도 90% 내외
- **medium 모델**: 속도 느림, 정확도 95% 내외
- **large 모델**: GPU 필수, 가장 정확한데 느림

CPU로 쓰시는 분은 small 모델 추천해요. 속도와 품질의 균형이 제일 좋아요.

## 실전: 3가지를 한 명령으로 🎬

블러 배경 + 중앙 크롭 + 자막 번인을 **한 번에** 처리하는 완전체 명령어입니다:

```bash
ffmpeg -i input.mp4 \
  -filter_complex "[0:v]crop=405:720:(iw-405)/2:0[cropped];[cropped]boxblur=10:10[blurred];[blurred]ass=subtitle.ass[final]" \
  -map "[final]" -map 0:a \
  -c:v libx264 -preset fast -crf 23 \
  output_reel.mp4
```

**동작 순서:**

1. 중앙 크롭 (16:9 → 9:16)
2. 블러 처리 (가장자리 뭉개기)
3. ASS 자막 번인

`--preset fast`는 인코딩 속도를 빠르게 하는 옵션이고, `--crf 23`은 화질 설정(숫자 작을수록 고화질, 기본 23)입니다.

그런데 사실 이 명령어 그대로 쓰시면 되는데, 자막 파일(subtitle.ass)이 미리 준비되어 있어야 해요. Whisper로 자막 생성하는 단계가 선행되어야 하거든요.

순서 정리하면:

```
[1] Whisper로 자막 생성 (SRT/ASS 파일)
[2] ffmpeg로 영상 변환 + 자막 번인
```

이 두 단계가 기본이고, 영상만 변환하고 싶다면 자막 번인 단계는 건너뛰면 돼요.

### 실제 사용 예시

제가 실제로 한 번 해본 과정을 공유할게요:

```bash
# 1단계: 자막 생성
whisper tutorial.mp4 --model small --output_format ass

# 2단계: 영상 변환 + 자막 번인
ffmpeg -i tutorial.mp4 \
  -vf "crop=405:720:(iw-405)/2:0,ass=tutorial.ass" \
  -c:v libx264 -preset fast -crf 23 \
  -c:a copy \
  tutorial_vertical.mp4
```

이렇게 하면 tutorial.mp4가 세로 영상으로 변환되고, 자막도 같이 번인돼요.

사실 이 예시를 그대로 따라하시면 됩니다. 파일명만 바꾸면 되거든요. 그리고 만약 영상이 1080p라면 crop 값도 1080 기준에 맞게 조절하시면 돼요. 1080p 기준으로는 너비가 607px이 돼요. 계산 공식은 위에서 설명드렸으니까 참고하세요.

영상을 변환한 다음에 결과를 확인하는 습관까지 들이면완벽해요. 변환한 영상이 자막이 깨져 보인다면, ASS 파일 경로가 잘못된 거예요. 자막이 아예 안 보인다면, ffmpeg가 자막을 지원하는 빌드인지 확인해보세요.

## 자주 쓰는 ffmpeg 명령어 레퍼런스

변환 말고도 ffmpeg는 많은 걸 할 수 있어요. 자주 쓰는 것만 정리하면:

```bash
# 영상 정보 확인
ffprobe -v error -show_entries stream=width,height,duration -of default=noprint_wrappers=1 input.mp4

# 영상 잘라내기 (30초부터 1분까지)
ffmpeg -i input.mp4 -ss 00:00:30 -to 00:01:00 -c copy output.mp4

# 영상 속도 2배 빠르게
ffmpeg -i input.mp4 -filter:v "setpts=0.5*PTS" -c:a copy output.mp4

# 영상 회전 (90도 시계방향)
ffmpeg -i input.mp4 -vf "transpose=1" -c:a copy output.mp4

# 영상 합치기 (filelist.txt 필요)
# filelist.txt 내용: file '1.mp4'\nfile '2.mp4'
ffmpeg -f concat -safe 0 -i filelist.txt -c copy output.mp4
```

사실 이 정도만 알아도 영상 편집의 80%는 해결돼요. 나머지 20%는 찾아보면 다 나와요.

특히 영상을 잘라내는 기능(`-ss`와 `-to`)은 정말 쓸모 많아요. 긴 영상의 특정 구간만잘라내서서 릴스로 만들 때 유용하거든요. 원본 영상의 30초부터 1분 사이만잘라내고 싶은 경우, `-ss 00:00:30 -to 00:01:00`으로 지정하면 돼요. `-c copy` 옵션은 영상을 다시 인코딩하지 않고 잘라내기만 해서 속도가 엄청 빠르다는 장점이 있어요.

영상의 속도를 조절하는 기능도 알아두면 좋아요. 예를 들어 2배 빠르게 하고 싶으면 `setpts=0.5*PTS`를 쓰면 되고, 0.5배 느리게 하고 싶으면 `setpts=2.0*PTS`를 쓰면 돼요. 자막이 있는 영상에서속도를 조절하면、자막도 같이조정되니까 자막 번인 전에 하면 좋아요.

## SNS 플랫폼별 권장 출력 스펙

변환한 영상을 어디에 올리느냐에 따라 설정이 달라져요.

| 플랫폼      | 비율          | 권장 해상도              | 자막 위치 |
| ----------- | ------------- | ------------------------ | --------- |
| 인스타 릴스 | 9:16          | 1080×1920                | 하단 중앙 |
| 유튜브 쇼츠 | 9:16          | 1080×1920                | 하단 중앙 |
| 틱톡        | 9:16          | 1080×1920                | 하단 중앙 |
| 트위터/X    | 16:9 또는 1:1 | 1920×1080 또는 1080×1080 | 하단      |

자막 위치는 플랫폼마다 다릅니다. 인스타 릴스나 쇼츠는 하단 중앙이 기본인데, 유튜브 쇼츠는 자막이 영상 위에 오버레이되는 형태라 위치가 좀 다를 수 있어요.

### 플랫폼별 자막 위치 조정

ASS 자막 파일을 만들 때 위치를 조절할 수 있어요:

```ass
[Script Info]
Title: Instagram Reels

[V4+ Styles]
Format: Name, Fontname, Fontsize, PrimaryColour, SecondaryColour, OutlineColour, BackColour, Bold, Italic, Underline, StrikeOut, ScaleX, ScaleY, Spacing, Angle, BorderStyle, Outline, Shadow, Alignment, MarginL, MarginR, MarginV, Encoding
Style: Default,Arial,48,&H00FFFFFF,&H000000FF,&H00000000,&H00000000,0,0,0,0,100,100,0,0,1,2,0,2,10,10,30,1

[Events]
Format: Layer, Start, End, Style, Name, MarginL, MarginR, MarginV, Effect, Text
Dialogue: 0,0:00:00.00,0:00:05.00,Default,,0,0,30,,자막 내용
```

`Alignment: 2`는 하단 중앙이에요. 5이면 하단 왼쪽, 1이면 하단 오른쪽이에요.

## 마무리

ffmpeg의 장점은 **한 번 파이프라인을 만들어 두면 매일 반복**할 수 있다는 것입니다.

자막 생성(Whisper) → 영상 변환(ffmpeg) → 인스타 업로드(API)까지 연결하면, 릴스 제작의 거의 모든 과정을 자동화할 수 있습니다. 가장 어려운 건 자막 생성이고, 그 다음은 ffmpeg 명령어 한 줄이 전부입니다.

솔직히 처음엔 명령어가 낯설었어요. 근데 막상 써보면 생각보다 안 무서워요. 사실... 저는 3번쯤 써보다가 익숙해졌어요.

자막 생성하는 단계에서 Whisper가 CPU를 많이 먹어서 오래 걸릴 수 있어요. GPU가 있는 분들은 인코딩 속도가 훨씬 빠르거든요. 영상이 길수록 이 속도 차이가 체감이 돼요. GPU가 있는 분들은 `--model small` 대신 `--model medium`으로 하시면 더 정확한 자막이 생성돼요.

**핵심 체크리스트:**

- [ ] ffmpeg 설치 (`brew install ffmpeg`)
- [ ] Whisper 설치 (`pip install whisper`)
- [ ] 입력 영상 해상도 확인 (ffprobe)
- [ ] 원하는 변환 방식 선택 (블러 배경 vs 중앙 크롭)
- [ ] 변환 명령어 실행
- [ ] 결과물 확인 (mpv나 VLC로 재생)
- [ ] SNS 플랫폼별 스펙 확인 후 필요시 재변환
